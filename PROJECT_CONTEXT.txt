1. THE IDEA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SkillMateAI is an EdTech platform built for the "Gemma 4 Good" hackathon (deadline
May 18 2026). The core concept: Gemma 4 AI acts as a silent teaching partner —
after a live tutoring session between a teacher and student, the teacher uploads
the session transcript and Gemma 4 automatically generates personalised quiz
questions based on what was actually taught.

The flow:
  Teacher creates session → teaches student via Zoom/Meet →
  uploads transcript (paste / .txt / .srt file) →
  Gemma 4 analyses transcript and generates 5 MCQ questions →
  student takes quiz → AI tracks progress and gives feedback

Key differentiator: questions are generated FROM the actual lesson content,
not from a generic topic. A Newton's Laws session generates questions about
the specific examples the teacher used, not a generic physics quiz.

Target users:
  - Private tutors (K-12, college)
  - Students who need structured post-session review
  - Grade levels: Elementary through College

  grade_level     String?  -- students only, e.g. "Grade 10", "College Freshman"

  id          String PK
  student_id  FK → profiles.id

  topic       String

  status      String  -- "pending" | "accepted" | "rejected"

Table: enrollments
  id          String PK
  student_id  FK → profiles.id
  teacher_id  FK → profiles.id
  request_id  FK → teacher_requests.id?
  created_at  DateTime(tz)

Table: sessions
  id               String PK
  teacher_id       FK → profiles.id
  student_id       FK → profiles.id?
  enrollment_id    FK → enrollments.id?
  topic            String
  status           String  -- "scheduled" | "completed" | "analysed"
  scheduled_at     DateTime(tz)?
  recording_link   String?
  transcript       Text?
  session_notes    Text?
  understanding_score  Int?
  weak_topics      JSON?
  created_at       DateTime(tz)
  ended_at         DateTime(tz)?

Table: questions
  id              String PK
  session_id      FK → sessions.id?
  student_id      FK → profiles.id?
  content         Text
  options         JSON  -- ["A) ...", "B) ...", "C) ...", "D) ..."]
  correct_answer  String  -- must match one option exactly
  explanation     Text
  topic           String
  difficulty      String  -- "easy" | "medium" | "hard"
  status          String default "approved"
  source          String default "session"  -- "session" | "transcript" | "self_study"
  created_at      DateTime(tz)

Table: quiz_attempts
  id          String PK
  student_id  FK → profiles.id
  session_id  FK → sessions.id?
  topic       String
  questions   JSON  -- [question_id, ...]
  answers     JSON  -- {question_id: "A) ..."}
  score       Int
  total       Int
  created_at  DateTime(tz)

6. AI INTEGRATION DETAILS (backend/services/gemma.py)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Model: gemma-4-26b-a4b-it via google-generativeai SDK
Config: temperature=0.2, response_mime_type="application/json"

Three functions:

1. generate_questions_from_transcript(transcript, topic)
   → Returns list of 5 dicts, each with:
     content, options[4], correct_answer, explanation, topic, difficulty
   → Used by POST /sessions/{id}/generate-from-transcript
   → Teacher pastes/uploads transcript in UI → questions saved to DB → student quizzes

2. analyse_session(transcript, topic)
   → Returns { understanding_score(0-10), weak_topics[], summary, questions[5] }
   → Used internally for full session analysis

3. generate_progress_feedback(attempts)
   → Returns 3-4 sentence plain-text feedback string
   → Called by GET /progress to give student encouragement + study tips

Critical implementation note — _extract_json():
  Gemma 4 outputs chain-of-thought reasoning before the actual JSON.
  response_mime_type="application/json" suppresses this in most cases,
  but not always. The fallback scans ALL positions of "[" and "{" in
  REVERSE ORDER and tries json.loads() at each — the last valid JSON
  block is the actual answer (not the thinking). rfind() alone is wrong
  because it finds inner brackets like "options": [ rather than the
  outer array start.

Transcript input formats supported (client-side):
  - Plain text paste into textarea
  - .txt file upload (read as-is)
  - .srt file upload (parseSrt() strips sequence numbers and timecodes)

10. KEY BUGS FIXED AND WHY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Dashboard blank page (two causes):
   a) Corrupted JWT in localStorage → getProfile() returned null → middleware
      redirected to /auth → middleware saw token → bounced back to /dashboard.
      Fix: call clearToken() in getProfile() whenever decode fails.
   b) null profile fell through to teacher dashboard code.
      Fix: explicit "if (!profile) return null" guard before role check.

2. Gemma 4 JSON parsing:
   - rfind("[") found inner brackets (e.g. inside "options": []) not the outer array.
   - Fix: scan ALL "[" and "{" positions in reverse, try json.loads() at each.

3. Supabase "@" in password URL-encoded:
   - "postgresql://postgres:@Chittu2026@..." is ambiguous.
   - Fix: encode "@" as "%40" in the connection string.

4. SQLAlchemy "postgres://" dialect:
   - SQLAlchemy 1.4+ dropped "postgres://" scheme (only "postgresql://").
   - Render/Heroku-style DATABASE_URL uses "postgres://".
   - Fix: DATABASE_URL.replace("postgres://", "postgresql://", 1) in database.py.

5. Render IPv6 → Supabase connection refused:
   - Render free tier connects via IPv6; Supabase direct host resolves to IPv6.
   - Fix: use Supabase Transaction Pooler (port 6543, IPv4 only).

6. CORS "Disallowed CORS origin" on OPTIONS returning 400:
   - Starlette CORSMiddleware passes preflight to router if origin not in allowlist.
   - Router returned 400, browser blocked request with no CORS headers.
   - Fix for hackathon: allow_origins=["*"] (no allow_credentials needed since
     auth uses Authorization header, not cookies).

7. vercel.json rootDirectory:
   - "rootDirectory" is a Vercel dashboard setting, not a vercel.json field.
   - Having it in vercel.json caused "Invalid request" deployment error.
   - Fix: set Root Directory to "frontend" in Vercel dashboard; remove from JSON.

8. Next.js security vulnerability:
   - 15.3.1 flagged as vulnerable by Vercel build.
   - Fix: upgraded to 15.5.18 (latest patch in 15.x line).
